Controlador
    Request pode conter:
        - ? LEVEL -ou- MAINSTAT VALUE
        - ? SUBSTATS adicionais (não pode alterar os existentes)
        - ? SUBSTAT VALUES (pode somente aumentar o valor de existentes ou de novos substats - em múltiplos permitidos)
        - ? CHAR
    Requisição pode conter infos opcionais desde que ela não tenha se alterado ou façam sentido de existir

    Passo-a-passo
        - Recebe a req e verifica se as informações estão no padrão esperado
            - Se não, Bad Request (400)
        - Chama o UpdArtifactDB com os dados recebidos
        - Se receber objeto error, retorna Bad Request InvalidParamError com o erro
        - Se UpdArtifactDB throws, retorna Internal Server Error (500)
        - Se receber true, retorna com status OK (200) e true

Data
    - Usa o ID da req para puxar as infos do artefato na base
    - Valida se dados alterados fazem sentido de existir
        - Se não, retorna InvalidParamError com o erro
    - Recalcula os valores de score default
    - Adiciona campo de "Date modified" com a data/hora da req
    - Se sim, Chama o Infra com estes novos campos somados à todas as infos básicas ajustadas do artefato
    - Se receber erro de infra, throw error
    - Se receber true, retorna true

Infra
    - Abre conexão com o MongoDB
    - Abre collection 'artifacts'
    - Atualiza ID do artefato com as infos recebidas
    - Se erro, throw error
    - Se ok, retorna true
